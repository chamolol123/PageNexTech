En progreso

Plataforma IoT ejecutándose en Raspberry Pi que orquesta sensores y actuadores mediante microservicios en Python (FastAPI). La modularidad permite variantes ligeras para ESP32 y despliegue por componentes en el borde.

78%

De dónde nace

Este desarrollo surge como tesis de estudiantes con el objetivo de validar una arquitectura IoT portable y mantenible: primero en laboratorio (Raspberry Pi) y luego en entornos embebidos de bajo consumo (ESP32). El foco es la orquestación desacoplada de dispositivos y reglas de negocio.

¿Por qué microservicios?

  • Portabilidad hacia ESP32: separar funciones en servicios pequeños habilita builds ligeras con lo estrictamente necesario para la MCU.
  • Escalabilidad y resiliencia: cada servicio (ingesta, reglas, notificaciones, etc.) se despliega y evoluciona de forma independiente.
  • Especialización por dominio: servicios con responsabilidades claras, facilitando pruebas y sustitución de componentes sin romper el sistema.

Tecnologías utilizadas (actuales)

  • Lenguaje / Framework: Python con FastAPI (todos los servicios).
  • Mensajería: MQTT como bus de eventos para telemetría y control.
  • Persistencia semántica: Ontologías OWL (archivos) para modelar dispositivos, estados y relaciones. Consultas/manipulación con librerías tipo RDFLib / OWLReady2. No usamos base de datos relacional.
  • Despliegue: Raspberry Pi como gateway; componentes seleccionados compilados/adaptados para ESP32 cuando se requiere ejecutar en el borde.
  • Integración de UI: La interfaz de usuario es una app móvil desarrollada por otro equipo que consume nuestra API; no tenemos front web propio.

Prototipo funcional

Fotografía del prototipo implementado en laboratorio. El gateway en Raspberry Pi coordina sensores, actuadores y módulos de comunicación; la API expone servicios para que la app móvil gestione reglas y estados.

Arquitectura (vista rápida)

  • Device Agent (Raspberry/ESP32): cliente MQTT para publicar telemetría y recibir comandos.
  • Ingesta (FastAPI): normaliza mensajes MQTT y actualiza el modelo semántico (OWL).
  • Reglas/Notificaciones (FastAPI): evalúa condiciones y emite eventos/acciones.
  • API de Integración (FastAPI): endpoints consumidos por la app móvil (no web).
  • Persistencia: ontologías OWL versionadas en archivos; sin BD relacional.
