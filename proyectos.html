En progreso

Plataforma IoT desplegada en Raspberry Pi que orquesta objetos inteligentes (sensores/actuadores) mediante servicios desacoplados. El diseño por microservicios permite ejecutar componentes de forma independiente —incluyendo variantes ligeras aptas para ESP32.

De dónde nace

El proyecto surge como tesis de estudiantes orientada a crear una solución IoT modular para monitoreo y control en tiempo real. La meta inicial fue validar una arquitectura extensible que facilite pruebas rápidas en laboratorio (Raspberry Pi) y posterior despliegue en hardware embebido de bajo consumo.

¿Por qué microservicios?

  • Portabilidad hacia ESP32: separar funciones en servicios pequeños permite crear builds ligeras (solo lo necesario) para ejecutarse en la ESP32 o cerca del borde.
  • Escalabilidad y resiliencia: cada servicio se despliega y escala de manera independiente, evitando cuellos de botella del monolito.
  • Mantenibilidad: dominios bien definidos (ingesta, reglas, notificaciones, APIs), ciclos de vida y stacks diferentes sin interferir.

Tecnologías utilizadas

  • Hardware: Raspberry Pi (gateway), sensores/actuadores; destino ligero: ESP32.
  • Mensajería: MQTT (broker), tópicos por dispositivo/servicio.
  • Back-end (servicios): Java/Spring Boot y/o Python (FastAPI) en contenedores.
  • Persistencia: PostgreSQL / SQLite según despliegue.
  • Orquestación: Docker Compose (desarrollo) y opción a K3s/Kubernetes (escala).
  • Observabilidad: logs estructurados, métricas y panel básico de eventos.
  • Front: app web para estado de dispositivos y reglas.

Prototipo funcional

A continuación se muestra una fotografía del prototipo implementado en laboratorio. El sistema utiliza una Raspberry Pi como nodo principal, con sensores, actuadores y módulos de comunicación coordinados mediante microservicios.

Arquitectura (vista rápida)

  • Device Agent: cliente MQTT en el borde (Raspberry/ESP32) para publicar telemetría y recibir comandos.
  • Ingesta: servicio que normaliza mensajes y valida esquema.
  • Reglas/Alerts: evaluación de umbrales y disparo de notificaciones.
  • API Gateway: expone endpoints para UI e integraciones.
  • UI Web: tablero de dispositivos, eventos y control básico.
